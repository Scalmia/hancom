정리 노트

마음에 드는 웹 사이트 분석

이 웹사이트 효과는 일반 이미지나 GIF가 아닌, ogl 이라는 WebGL 라이브러리로 GPU가 매 프레임마다 수학 공식을 사용해 실시간으로 그려 내는 화면이다. 별 하나하나가 미리 그려진 그림이 아니라, 이 좌표에 별이 있을 확률/색깔/밝기를 계산하는 함수로 즉석에서 계산되는 것이다.

3D 그래픽을 웹사이트에 담아내고자 하려 했던 난, 언젠가 WebGL 라이브러리를 사용하게 될 것이라고 생각하고 있었고, 이 웹사이트는 그 출발선에 서기 전 맛보기 용으로 훌륭한 교보재였다.

CLAUDE 를 활용하여 이 효과의 코드를 분석했을 때, 핵심 기법 4가지 결과를 얻을 수 있었다.

  1. 레이어 깊이감 표현 (Parallax) : 별을 4개 층(NUM_LAYER = 4.0)으로 나눠서, 층마다 크기와 이동 속도를 다르게 주었다. 예를 들어 가까운 층은 크고 빠르게, 먼 층은 작고 느리게 움직이도록. 이를 통해 우주 공간의 원근감을 흉내내었다고 한다.
  2. 의사 난수(Pseudo-random) 배치 : Hash21 함수는 좌표를 넣으면 항상 같지만 무작위처럼 보이는 숫자를 뱉어낸다. 그래서 매 프레임 다시 계산해도 별의 위치는 흔들리지 않고 고정돼있으면서, 사람 눈엔 자연스럽게 흩뿌려진 것처럼 보인다고 한다.
  3. 반짝임(Twinkle)과 색조(Hue) 조정 : 시간에 따라 삼각파 함수(trisn)로 별의 밝기를 미세하게 진동시켜서 반짝이는 느낌을 주고, HSV 색상 모델로 전체 별의 색조(uHueShift)를 한 번에 통일감 있게 조정했다고 한다.
  4. 마우스 반응(Interactivity) : 마우스 위치를 셰이더에 실시간으로 넘겨서(uMouse), 별들이 마우스를 밀어내듯 좌표가 휘어지는 효과(repulsion)를 주었다. 사용자 행동에 반응하는 동적인 배경을 만드는데 핵심인 요소이다.

이런 효과를 만약 <div> 수백 개로 별을 하나씩 만들고 JS로 애니메이션을 만들었다면 브라우저가 많이 버벅였을 것이다. 근데 이건 GPU가 화면의 픽셀 하나하나를 병렬로 계산하기 때문에 부드러운 60프레임으로 돌아가는 것이다.

아직은 어떤 방식을 쓴것이고 내가 무엇을 이해하고 넘어가야 하는지 눈에 들어오진 않는다. 다만, 현재 내가 공부중인 프론트엔드 문법조차 처음 봤을 때는 도대체 무슨 말을 하는지 전혀 이해가 안됐던것을 미루어봤을 때, 이를 이해하고 나만의 방식으로 습득하는 것은 시간문제라고 생각한다. 또한 이런 화려한 효과들을 자유자재로 생성하고 수정하고 꾸며낼 수 있게 된다면, 내가 만드는 프로그램과 웹사이트에 동적인 효과를 자연스럽게 포함시켜 독특하고 세련된 UX를 체험시켜 줄 수 있을 것이라고 생각한다.

HTML 정리

웹을 이루는 세 가지 언어

웹페이지 하나는 HTML, CSS, JS 세 가지가 모여 만들어진다. 흔히 레고로 집을 짓는 것에 비유되는데, HTML은 집에 무엇이 있는지(글, 사진, 버튼)를 나타내는 뼈대이고, CSS는 그것이 어떻게 보이는지(색, 크기, 위치)를 정하는 꾸미기 작업이며, JS는 그것이 어떻게 움직이는지(클릭, 반응)를 담당하는 전기 배선과 스위치에 해당한다.

기본 문서 구조

  • 모든 HTML 문서는 <!DOCTYPE html> 선언으로 시작하고, <html lang="ko">로 언어를 지정한다.
  • <head>는 화면에 직접 보이지 않는 정보를 준비하는 영역이다. <meta charset="UTF-8">로 한글이 깨지지 않도록 인코딩을 지정하고, <title>도 이 안에 들어간다.
  • <body>는 실제 화면에 보이는 내용을 작성하는 영역이다.
  • 태그는 열었으면 반드시 닫아야 하며, 여는 태그·내용·닫는 태그를 합쳐 하나의 "요소(element)"라고 부른다.
  • 주석은 <!-- 이건 주석입니다 --> 형식으로 작성한다.
  • 태그 이름은 대소문자를 구분하지 않는다.

엔티티(Entity)

<, >, & 기호는 HTML이 명령어로 착각할 수 있기 때문에, 글자 그대로 표시하려면 약속된 엔티티로 바꿔 써야 한다.

  • &lt; → <
  • &gt; → >
  • &amp; → &

이미지와 링크

  • <img> 태그는 src(이미지 주소), alt(대체 텍스트), width/height(크기) 속성을 갖는다.
  • <a href="...">는 클릭 시 다른 곳으로 이동하는 링크를 만든다. target="_blank"를 지정하면 새 창(탭)에서 열린다.
  • href="#id이름"으로 문서 내 특정 id 요소로 이동할 수 있다(class로는 불가능하다 — 한 문서에 여러 개가 있을 수 있기 때문).
  • mailto:로 이메일 전송을, tel:로 전화 걸기를 연결할 수 있다.

시맨틱 태그

  • 페이지는 보통 header, main, footer로 큰 틀을 나눈다.
  • 블록 요소(p, h1, ul, header 등)는 한 줄을 통째로 차지하고, 인라인 요소(a, strong, em 등)는 글자 사이에 끼어드는 방식으로 배치된다.
  • ul/ol은 목록을 담는 상자 역할을, li는 그 안의 개별 항목을 만드는 역할을 한다.
  • nav는 메뉴·길 안내 같은 상단 내비게이션 역할을 한다.
  • section은 하나의 주제를 설명하는 덩어리를 나타낸다.
  • article은 글, 뉴스, 카드처럼 그 자체로 완결된 콘텐츠를 나타내며, 검색엔진과 스크린 리더가 우선적으로 인식한다.
  • aside는 사이드바나 광고 영역을 나타낸다.
  • 이런 시맨틱 태그를 쓰는 이유는 검색엔진과 화면 낭독기가 문서 구조를 더 잘 이해하도록 해서, div만 쓰는 것보다 SEO와 접근성 면에서 유리하기 때문이다.

자기소개 페이지 만들어보기

줄바꿈과 구분선

  • <br>은 한 문단 안에서 줄을 바꾸고, <hr>은 내용을 구분하는 가로줄을 그린다. 둘 다 닫는 태그가 없는 빈 요소다.

표(table)

  • table 태그 안에서만 쓸 수 있는 전용 태그들이 있다. th는 가운데 정렬과 굵은 글씨가 적용되는 제목 칸, td는 일반 데이터 칸이다.
  • border 값이 0이면 테두리가 없고, 값이 커질수록 테두리가 두꺼워진다.
  • thead/tbody 내부에서 colspan, rowspan으로 칸을 병합할 수 있다. 단, 병합을 쓰더라도 실제 tr과 값은 존재해야 한다.

폼(form)

  • form 안에서 label로 이름표를 달고 input으로 입력 칸을 만든다. button type="submit"으로 모든 데이터를 한 번에 전송할 수 있다.
  • button은 form 밖에서는 onclick 같은 별도 이벤트로만 동작한다.
  • email은 @ 포함 여부를 검사하고, password는 입력 글자를 가려준다.
  • checkbox는 여러 개 동시 선택이 가능하고, radio는 같은 name을 가진 것들 중 하나만 선택된다.
  • date는 날짜 입력을, textarea는 크기 조절 가능한 여러 줄 입력을 제공한다.
  • select는 드롭다운을 만들고, 내부 option에 value를 지정해 값의 순서를 안정적으로 유지할 수 있다.

미디어 삽입하기

  • iframe 태그로 유튜브 영상 등 다른 페이지의 콘텐츠를 그대로 가져와 삽입할 수 있다. 유튜브의 "퍼가기" 버튼으로 제공되는 코드를 그대로 붙여넣으면 된다.
  • 구글 지도도 같은 방식으로 삽입할 수 있으며, src="...output=embed" 형태로 직접 작성할 수도 있다. 주소에서 조건을 이을 때는 &를 &amp;로 이스케이프해야 한다.
  • <video controls src="..."></video>로 직접 보유한 영상 파일을 재생할 수 있다.
  • source 태그를 여러 개 나열하면 브라우저가 재생 가능한 형식을 자동으로 선택하고, 파일 형식(MIME type)을 함께 명시하는 것이 권장된다.
  • 재생 가능한 형식이 하나도 없을 경우 표시할 대체 문구도 작성할 수 있다.

data, datalist, 정의 목록

  • <data value="2024-06-01">2024년 6월 1일</data>처럼, value에는 컴퓨터가 읽을 값을, 태그 내용에는 사람이 읽을 글자를 넣는다.
  • input의 list 속성과 datalist의 id를 매치시키면, 입력 칸에 추천 키워드 목록을 띄울 수 있다.
  • label의 for와 input의 id를 같게 지정하면, 글자를 클릭해도 입력 칸을 클릭한 것과 같은 효과를 준다.
  • dl/dt/dd는 국어사전처럼 용어와 그 뜻을 짝지어 작성할 수 있는 태그다. dt는 용어, dd는 그에 대한 설명이며 들여쓰기 되어 표시된다.

class와 id

  • class는 요소에 이름표를 붙여 CSS/JS에서 선택할 수 있게 해주는 속성이다. 공백으로 구분해 하나의 요소에 여러 클래스를 동시에 적용할 수 있다.
  • id는 문서 안에서 단 한 번만 지정할 수 있고, 공백으로 여러 개를 동시에 부여할 수 없다. CSS 우선순위도 class보다 높다.

div와 span

  • div는 그 자체로는 의미가 없는(non-semantic) 태그지만, 여러 요소를 하나로 묶어 한 번에 스타일을 적용할 때 유용하다.
  • span도 non-semantic 태그이지만, 한 줄 전체가 아니라 글자 몇 개만 작게 감싸는 용도로 쓰인다. 가능하면 의미에 맞는 시맨틱 태그를 먼저 쓰고, 마땅한 게 없을 때 div/span을 쓰는 것이 권장된다.

파일 경로 지정하기

  • src(source)로 파일 경로를 지정한다. .은 현재 폴더, ..은 상위 폴더를 의미하며, 경로를 잘못 지정하면 이미지가 깨져서 나온다.

글자 꾸미기 태그

  • em은 이탤릭체, strong/b는 볼드체, u는 밑줄, s는 취소선, mark는 형광펜 표시, small은 작은 글씨, code는 코드체를 나타낸다.
  • blockquote는 긴 인용문을, q는 짧은 인용문을 나타내며, cite로 인용 출처 주소를 명시할 수 있다.
  • sub는 아래첨자, sup는 위첨자를 만든다.

접고 펼치기 (details)

  • details는 접거나 펼 수 있는 상자를 만들고, summary는 항상 보이는 제목을 지정한다.

Emmet 자동완성

  • ! 입력 후 Tab을 누르면 기본 HTML 뼈대가 자동 생성된다.
  • ul>li*3처럼 >는 안에 넣기, *n은 n번 반복을 의미한다.
  • .이름은 class, #이름은 id를 만들고, {글자}는 내용을 채운다.
CSS 정리

CSS 기본 문법과 연결 방법

CSS는 페이지에 옷을 입히는 것과 같다. 밋밋한 화면의 색, 크기, 위치를 보기 좋게 바꿔주는 역할이다. 한 줄의 짜임은 h1 { color: red; }처럼 선택자(누구를), 속성(무엇을), 값(어떻게)으로 이루어진다.

  • CSS는 styles/main.css에 작성한 뒤, HTML의 <head>에서 <link rel="stylesheet" href="...">로 불러온다.
  • 외부 방식: <link>로 연결하며, 여러 페이지에서 공유하고 관리하기 쉽다.
  • 내부 방식: <style> 태그를 head 안에 직접 작성하며, 한 페이지에만 적용된다.
  • 인라인 방식: 태그에 style="..."을 직접 넣으며, 그 요소 하나에만 적용된다.

코드 작성 순서 관례

  • 일반적인 것에서 구체적인 것 순서로 작성하는 것이 관례다: body(전체) → 요소(h1, p) → 클래스(.special) → 자손(li em) 순서로 쌓아 올린다.

선택자

  • :root는 문서의 최상위 요소를 가리키며, html과 같은 위치를 가지지만 명시도가 더 높다(가상 클래스 선택자로 10점, 겹치면 root가 이긴다). 전역 변수를 선언하는 관용적인 자리로도 쓰인다.
  • a 태그의 상태별 스타일을 지정하는 의사 클래스가 있다: link(기본 상태), visited(방문 후), hover(마우스를 올렸을 때), active(누르는 중). 이 순서를 지켜 작성해야 하며, 앞 글자를 따 LVHA라고 부른다.
  • class와 id는 가상 클래스가 아니다. :hover, :root, :nth-child()처럼 HTML에 직접 선언한 적 없는 상태나 구조적 사실을 브라우저가 실시간으로 판단해 골라주는 것이 가상 클래스다.
  • ::before, ::after, ::first-letter처럼 요소 안에 존재하지 않던 새로운 부분을 만들어내는 것은 가상 요소(pseudo-element)라고 하며, 가상 클래스와 구분하기 위해 콜론을 두 개(::) 쓴다.

박스 모델과 기본 속성

  • font-family는 글씨체를 지정하며, 여러 개를 나열하면 그 순서가 우선순위가 된다.
  • margin은 요소 바깥 여백을, padding은 콘텐츠와 테두리 사이 안쪽 여백을 조정한다. 값을 2개 쓰면 각각 위/아래, 왼쪽/오른쪽 순서다.
  • margin에 auto를 주면 요소를 가운데 정렬할 수 있는데, 이때 요소의 width가 지정되어 있어야 한다.
  • padding을 늘리면 콘텐츠 크기는 그대로인데 그 주변 여백이 두꺼워지면서 박스의 전체 바깥 크기가 커진다. 이 현상을 막으려면 box-sizing: border-box;를 지정해, padding과 border를 포함한 크기로 width/height를 고정할 수 있다. 실무에서는 거의 예외 없이 이 설정을 쓴다.
  • border는 두께, 모양(solid, dashed, dotted, double 등), 색상 순서로 지정한다.
  • text-align은 인라인 요소를 가로 기준으로 정렬한다.
  • margin-top은 요소 자체를 위쪽 바깥 여백으로 밀어내는 방식이며, 안쪽 여백인 padding-top과는 동작 방식이 다르다.
  • font-weight는 글자 굵기를, font-style은 이탤릭 등 글자 스타일을 조정한다.
  • color는 순수한 색상값만 받을 수 있어 linear-gradient는 적용되지 않는다(이미지 타입으로 분류되기 때문). 그라디언트를 쓸 수 있는 건 background(또는 background-image)뿐이다.

레이아웃: position, display, flex, grid

  • position은 요소가 문서 흐름에서 어떻게 배치되고 기준점이 어디인지를 정한다.
  • relative는 두 가지 역할을 한다 — top/left 등을 함께 쓰면 원래 자리를 차지한 채 시각적으로만 이동하고, 자식 요소가 absolute를 쓸 때 그 기준점이 되어준다.
  • absolute는 문서 흐름에서 완전히 빠져나와 가장 가까운 relative 조상을 기준으로 좌표를 잡는다(조상이 없으면 뷰포트 기준).
  • relative — 원래 자리 기준 이동
  • absolute — 위치 지정 조상 기준
  • fixed — 화면(뷰포트) 고정
  • sticky — 스크롤 시 화면에 달라붙음
  • inset은 top/right/bottom/left: 0;을 한 번에 쓰는 축약형으로, absolute와 함께 쓰면 부모와 똑같은 크기로 꼭 맞게 채워진다.
  • display는 요소의 배치와 크기 규칙을 결정한다. none은 화면에서 사라지고, block은 한 줄을 통째로 차지하며 width/height/margin/padding이 모두 정상 작동한다. inline은 글자처럼 옆으로 흐르되 width/height가 무시되고, inline-block은 옆으로 흐르면서도 크기 지정이 가능한 절충안이다.
  • flex: 1;은 flex-grow: 1; flex-shrink: 1; flex-basis: 0%; 세 속성을 한 번에 줄인 것이다. grid의 fr과 비슷하지만, grid는 부모가 틀을 미리 짜는 방식이고 flex는 자식들이 각자 몫을 요구하는 방식이라는 접근 차이가 있다.
  • flex 컨테이너로 만들지 않으면 자식은 그냥 display: block 상태로 남는다.

애니메이션과 관련된 개념들

  • vh/vw는 각각 뷰포트 세로/가로 크기를 기준으로 하는 단위다.
  • perspective는 관찰자와 화면 사이의 거리를 나타내며, 값이 작을수록 원근 왜곡이 심하게(입체적으로) 보인다.
  • 자주 쓰는 transform 함수로 translate, scale, rotate, skew가 있으며, 실무에서는 앞의 세 가지가 특히 많이 쓰인다.
  • ease는 시간에 따른 변화 속도를 정하는 곡선(cubic-bezier)이다. linear는 등속, ease(기본값)는 천천히 시작해 빨라졌다가 천천히 마무리, ease-in은 천천히 시작해 급하게 끝나고, ease-out은 빠르게 시작해 부드럽게 끝난다.
  • CSS 좌표계는 수학 그래프와 달리 아래로 갈수록 y값이 커지므로, 위로 떠오르는 애니메이션을 만들려면 translateY에 음수 값을 줘야 한다.
  • animation-iteration-count로 반복 횟수(숫자 또는 infinite)를 지정할 수 있다. transition에는 이 개념이 없는데, A에서 B로 한 번만 진행되는 것이기 때문이다.
  • linear-gradient(각도, 시작색, 끝색)은 두 색을 지정한 각도로 부드럽게 이어 배경으로 깔아준다(0deg는 아래→위, 90deg는 왼쪽→오른쪽 기준).

변수와 리셋

  • CSS 변수는 반드시 --로 시작해야 하며(--이름: 값;), 사용할 때는 var(--이름)으로 불러온다.
  • 브라우저마다 기본 margin, padding, box-sizing이 달라 CSS 리셋이 필요하다. box-sizing: border-box;는 실무에서 거의 항상 쓰지만, margin: 0; padding: 0;을 무조건 다 지우는 방식(Eric Meyer's Reset 같은 고전적 방식)보다는, normalize.css나 모던 CSS 리셋, 혹은 Tailwind/Bootstrap 같은 프레임워크 내장 리셋을 쓰는 것이 요즘 추세다.

Stylelint

  • CSS 전용 린터(맞춤법 검사기)로, 오타·잘못된 속성·중복·순서를 검사해준다(HTML은 대상이 아니다). extends로 기본 규칙 묶음을, plugin으로 추가 기능(순서 검사 등)을 가져오고, rules로 개별 규칙을 켤 수 있다.
  • npm으로 설치한 뒤 npx stylelint 명령어로 실행한다.

실전 프로젝트: 3D 카드 뒤집기

  • transition은 값이 바뀔 때 즉시 반영하지 않고 부드럽게 보간하는 속성이며, 지정한 속성에만 적용된다.
  • transform-style은 자식 요소들을 자신과 같은 3D 공간에서 취급할지 정하는 속성으로, flat과 preserve-3d 두 값만 있다. flat으로 두면 "뒷면"이라는 개념 자체가 사라져 뒷면 요소가 무시된다.
  • backface-visibility는 회전 각도가 -90°~90° 사이면 앞면이 보이는 상태, 90°~270° 사이면 뒷면이라 숨겨지는 상태로 판단한다.
  • hover를 감지할 때 회전하는 요소가 아닌 고정된 부모 요소(.card:hover .inner)를 기준으로 삼는 이유는, 회전 중인 요소 자체가 완전히 얇아지는 순간 마우스가 hover 상태를 놓쳐 깜빡이는 버그가 생길 수 있기 때문이다.

실전 프로젝트: hover 회전 + 확대, 자동 애니메이션, 네온 카드

  • transform: rotate() scale()처럼 기울이기와 확대를 함께 적용할 수 있다.
  • @keyframes로 동작을 정의한 뒤 animation으로 반복시킨다.
  • linear-gradient로 두 색의 그라디언트 배경을 만들고, box-shadow의 흐림 반경을 크게 줘서 네온 글로우 효과를 낸다.

실전 프로젝트: Grid와 반응형

  • grid-template-columns로 열을 만들고, 그 안에 카드를 배치해 갤러리를 구성한다.
  • 화면 너비가 줄어들면 @media 쿼리로 정렬 방향을 바꿔, 모바일과 PC 환경을 구분해 대응한다.

실전 프로젝트: 가짜 요소 활용

  • ::before, ::after는 선택자 하나당 딱 1개씩만 만들 수 있으며, 요소 전체에 붙는 고정된 장식을 만든다. HTML은 콘텐츠와 기능 설명에만 집중하고, 장식은 CSS가 전담하도록 역할을 나눠준다.
